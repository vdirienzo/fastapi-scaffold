Quick Start Guide

Levanta tu aplicación FastAPI completa con autenticación JWT, PostgreSQL, Redis y Docker en solo unos minutos.

Tiempo estimado: ~5 minutos

Setup Automatizado

La forma más rápida de comenzar. Un solo comando configura todo por ti:

bash 
./setup.sh

El script automáticamente:

  • Instala todas las dependencias con uv
  • Crea el archivo .env desde el template
  • Configura los pre-commit hooks
  • Inicializa el repositorio git
Recomendado

El setup automatizado es la opción recomendada para la mayoría de casos. Solo sigue con el setup manual si necesitas control granular sobre cada paso.

Setup Manual

1

Instalar Dependencias

Primero, asegúrate de tener uv instalado (el gestor de paquetes ultra-rápido para Python):

bash 
# Instalar uv si no lo tienes
curl -LsSf https://astral.sh/uv/install.sh | sh

# Instalar dependencias del proyecto
uv sync --all-extras
2

Configurar Variables de Entorno

Copia el archivo de ejemplo y personaliza los valores:

bash 
# Copiar archivo de ejemplo
cp .env.example .env

# Editar con tus valores
nano .env

Valores mínimos requeridos en .env:

env 
DATABASE_URL=postgresql+asyncpg://postgres:postgres@localhost:5432/fastapi_db
SECRET_KEY=your-super-secret-key-min-32-characters-here
3

Iniciar Servicios

Levanta PostgreSQL y Redis con Docker Compose:

bash 
# Solo base de datos y Redis
docker-compose up -d db redis

# O todos los servicios (incluyendo la app)
docker-compose up
4

Ejecutar la Aplicación

Inicia el servidor de desarrollo con hot-reload:

bash 
# Modo desarrollo (con auto-reload)
uv run uvicorn fastapi_project.main:app --reload

# O usando Make
make run

Verificar Instalación

1. Health Check

Verifica que el servidor esté corriendo correctamente:

bash 
curl http://localhost:8000/api/v1/health

Respuesta esperada:

json 
{"status": "healthy"}

2. Documentación Interactiva

Abre en tu navegador: http://localhost:8000/docs

3. Ejecutar Tests

bash 
# Tests básicos
make test

# Tests con coverage
make test-cov

Crear Tu Primer Usuario

Hay múltiples formas de crear un usuario. Elige la que prefieras:

  1. Abre http://localhost:8000/docs
  2. Busca el endpoint POST /api/v1/users
  3. Click en "Try it out"
  4. Ingresa los datos del usuario:
json 
{
  "email": "user@example.com",
  "username": "myuser",
  "password": "SecurePass123",
  "full_name": "My Name"
}
  1. Click en "Execute"
bash 
curl -X POST http://localhost:8000/api/v1/users \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "username": "myuser",
    "password": "SecurePass123",
    "full_name": "My Name"
  }'
bash 
http POST http://localhost:8000/api/v1/users \
  email=user@example.com \
  username=myuser \
  password=SecurePass123 \
  full_name="My Name"

Login y Obtener Token JWT

Una vez creado el usuario, obtén tokens de autenticación:

bash 
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "myuser",
    "password": "SecurePass123"
  }'

Respuesta con tokens:

json 
{
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "refresh_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "token_type": "bearer"
}
Tip

Guarda el access_token — lo necesitarás para autenticarte en endpoints protegidos. El refresh_token sirve para obtener nuevos access tokens cuando expiren.

Usar Endpoints Protegidos

Para acceder a endpoints que requieren autenticación, incluye el token en el header Authorization:

bash 
# Obtener información del usuario actual
curl http://localhost:8000/api/v1/users/me \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"

Respuesta esperada:

json 
{
  "id": 1,
  "email": "user@example.com",
  "username": "myuser",
  "full_name": "My Name",
  "is_active": true,
  "is_superuser": false,
  "created_at": "2025-01-04T12:00:00Z"
}

Variables de Entorno

Referencia completa de todas las variables de configuración disponibles:

Variable Descripción Default Requerida
DATABASE_URL URL de conexión a PostgreSQL (asyncpg) — Sí
SECRET_KEY Clave secreta para JWT (mínimo 32 chars) — Sí
REDIS_URL URL de conexión a Redis redis://localhost:6379 No
ACCESS_TOKEN_EXPIRE_MINUTES Tiempo de expiración del access token 30 No
REFRESH_TOKEN_EXPIRE_DAYS Tiempo de expiración del refresh token 7 No
CORS_ORIGINS Orígenes permitidos (separados por coma) http://localhost:3000 No
LOG_LEVEL Nivel de logging INFO No
DEBUG Modo debug (true/false) false No
ENVIRONMENT Entorno (development/staging/production) development No

Troubleshooting

Soluciones a problemas comunes:

Problema: No se puede conectar a PostgreSQL

Solución:

bash 
# Verificar si los servicios están corriendo
docker-compose ps

# Reiniciar base de datos
docker-compose restart db

# Ver logs de la base de datos
docker-compose logs db

Problema: El puerto 8000 ya está en uso

Solución:

bash 
# Encontrar proceso usando el puerto
lsof -i :8000

# Matar el proceso
kill -9 <PID>

# O usar un puerto diferente
uv run uvicorn fastapi_project.main:app --reload --port 8001

Problema: Los tests fallan con errores de base de datos

Solución:

bash 
# Asegurarse de que test database esté configurada
export DATABASE_URL=sqlite+aiosqlite:///:memory:

# Ejecutar tests
make test

Problema: Los pre-commit hooks fallan al hacer commit

Solución:

bash 
# Ejecutar hooks manualmente para ver errores
make pre-commit

# Arreglar issues y volver a intentar
make lint-fix
make format
git add .
git commit -m "fix: your message"

Tips de Seguridad para Producción

Importante

Estas configuraciones son críticas para ambientes de producción. No las ignores.

✅ Checklist de Seguridad

  • SECRET_KEY: Genera una clave aleatoria de al menos 64 caracteres usando openssl rand -hex 32
  • DEBUG: Asegúrate que DEBUG=false en producción
  • CORS: Configura CORS_ORIGINS solo con dominios permitidos específicos
  • HTTPS: Siempre usa HTTPS en producción. Configura un reverse proxy (Nginx/Traefik)
  • Rate Limiting: Habilita rate limiting para prevenir ataques de fuerza bruta
  • Database: Usa contraseñas fuertes y conexiones SSL para PostgreSQL
  • Tokens: Reduce ACCESS_TOKEN_EXPIRE_MINUTES a 15-30 minutos
  • Headers: Configura security headers (HSTS, CSP, X-Frame-Options)

🔐 Generación de SECRET_KEY

bash 
# Generar una clave segura
openssl rand -hex 32

# O con Python
python -c "import secrets; print(secrets.token_urlsafe(64))"

🛡️ Headers de Seguridad Recomendados

nginx.conf 
# En tu configuración de Nginx
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header Content-Security-Policy "default-src 'self'" always;
